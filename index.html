Store Master

List every store to be monitored. Output: Store_Master.csv

  2. Upload Manual stores separately if needed — Store Type = Manual, Test Shop = 0.
  3. Upload Test Stores separately if needed — Test Shop = 1 set automatically.

SKU Assortment

Define the full SKU universe first, then build assortments as subsets. Output: SKU_Assortment.csv

Two-phase workflow:
  1. Phase 1 — SKU Universe: Upload ALL Self SKUs and Competitor SKUs once. These define the flag defaults for the entire project. Select reserved classes (Others is mandatory).
  2. Phase 2 — Assortments: Each assortment is a subset of the universe (typically Self SKUs for a specific store type). Upload just the SKU names — the tool auto-fills flags from the universe and auto-adds competitors and reserved classes for matching categories.

Store-Assortment Mapping

Link each store to a product assortment and shelf layout. Output: Store_Assortment_Mapping.csv

ℹ️Stores are pulled from Store Master (Tab 2). Categories are pulled from SKU Assortment (Tab 4). Fill those tabs first for best results — but you can also add rows manually below.
How to fill this tab:
  1. Define Shelf Sections and group them into Shelf Combinations. Each combination has a name (internal label), a Category Heading (exported to SAM — defaults to "Primary Placement"), and the shelf sections that belong to it.
  2. Assign each store an assortment and a combination in the Store Assignment table.
  3. Click "Generate SAM Rows" — one row is created for every Store × Category pair, with shelf sections auto-filled from the combination.
  4. Review the output table and confirm when ready.
📐 Shelf Sections & Combinations

Step 1 — define individual shelf sections (with optional image links). Step 2 — create named combinations, set the Category Heading for each (defaults to "Primary Placement"), and check which sections belong to it. The Category Heading is exported to the SAM file; the combination name is just the internal label used in store assignment.

Validate & Download

Check all files for errors, then export as a ZIP.

Final steps:
  1. Run Validation first — required before any file can be downloaded. Errors and warnings appear here and on each file's card below.
  2. Red BLOCKER errors must be fixed before uploading to MDM. Individual files are still downloadable; the full ZIP is blocked until all blockers are resolved.
  3. Yellow WARNING items are non-critical — review before uploading, but downloads are not blocked.
  4. Expand "QC guidelines" on each card to verify the file meets MDM upload requirements before sending.
  5. Download individual files or click "Download All as ZIP" to export all 6 MDM files at once.
