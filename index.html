Store Master

List every store to be monitored. Output: Store_Master.csv

What to do on this tab:
  1. Upload Digital stores — Store Type = Digital, Test Shop = 0 set automatically.
  2. Upload Manual stores separately if needed — Store Type = Manual, Test Shop = 0.
  3. Upload Test Stores separately if needed — Test Shop = 1 set automatically.
  4. Only sections with data are included in the output. If only Digital stores are added, only those are exported.
  5. All lists are combined in the grid below. Fix red rows before downloading.

SKU Assortment

Define the full SKU universe first, then build assortments as subsets. Output: SKU_Assortment.csv

Two-phase workflow:
  1. Phase 1 — SKU Universe: Upload ALL Self SKUs and Competitor SKUs once. These define the flag defaults for the entire project. Select reserved classes (Others is mandatory).
  2. Phase 2 — Assortments: Each assortment is a subset of the universe (typically Self SKUs for a specific store type). Upload just the SKU names — the tool auto-fills flags from the universe and auto-adds competitors and reserved classes for matching categories.
📋 Column reference — SKU Assortment ▼ hide

Regular OSARequiredSet automatically from universe defaults: 1 for Self, 0 for Competitor/Reserved (unless competition OSA tracking is on).
SOSRequiredDefault 1 for all SKUs.

Store-Assortment Mapping

Link each store to the right product assortment. Output: Store_Assortment_Mapping.csv

⚠️Complete Store Master and SKU Assortment first. Stores and assortment names are pulled from those tabs.
What to do on this tab:
  1. The Smart Assortment Assigner below shows all your assortments (from SKU Assortment tab). Click any assortment block to expand it.
  2. Check the checkbox next to each store that should use that assortment. Fill in Category Name and Shelf Section.
  3. Rows appear automatically in the table below as you check stores — no need to type them manually.
  4. If you added stores or assortments after opening this tab, click "Refresh" to reload the list.
🔗 Smart Assortment Assigner
Assortment names are pulled from SKU Assortment (Tab 4). Stores are pulled from Store Master (Tab 2). Expand each assortment block and check the stores that should use it.
